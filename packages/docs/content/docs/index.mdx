Jeremy is a documentation RAG (Retrieval-Augmented Generation) service that ingests library documentation and makes it queryable through semantic search, so AI coding assistants can look up accurate, up-to-date docs on demand.

## Why Jeremy?

AI assistants have knowledge cutoffs and can't access the latest documentation for every library. Jeremy solves this by:

- **Ingesting docs from any source** -- point it at an `llms.txt` file or a documentation URL, and Jeremy fetches, chunks, and embeds the content.
- **Semantic search via vector embeddings** -- queries return the most relevant documentation chunks, not just keyword matches.
- **Serving context to AI assistants** -- the MCP server integrates directly with Claude Code and other MCP-compatible tools, giving them access to your indexed docs.

## Key Features

- **MCP Server** -- an MCP (Model Context Protocol) server that AI assistants like Claude Code can use to query documentation in real time.
- **Dashboard** -- a web UI for managing your account, API keys, and libraries.
- **REST API** -- programmatic access to search, context retrieval, ingestion, and library management.
- **200+ pre-indexed libraries** -- popular frameworks and tools are already indexed and ready to search.

## How It Works

1. **Sign up** -- create an account at [jeremy.khuur.dev](https://jeremy.khuur.dev).
2. **Connect** -- run one command to register the MCP server with Claude Code.
3. **Search** -- when an AI assistant needs documentation, it calls Jeremy's MCP tools. Jeremy performs a semantic vector search and returns the most relevant chunks.
